# プロジェクト設計書：Strudel AI拡張機能

## 1. プロジェクト概要

### 1.1. 目的

Webベースのライブコーディング環境「Strudel」のフロントエンドを改造し、AIによるコード生成支援機能と、静的なコードテンプレート機能を追加する。

### 1.2. 開発アプローチ

Codeberg上のStrudel公式リポジトリをフォークし、ソースコードを直接編集する。最終的に、完成したアプリケーションを公開Webサービスとしてデプロイすることを目指す。

### 1.3. 使用技術

- **フレームワーク:** Astro
- **パッケージ管理:** pnpm
- **デプロイ先:** Netlify等のサーバーレスプラットフォーム

---

## 2. 実装仕様

### 2.1. AIチャット機能

- **UI:**
    - 画面右下にフローティングボタンを配置し、クリックでチャットウィンドウを開閉する。
- **バックエンド:**
    - OpenAIのモデル（`gpt-4.1`等を想定）を利用する。
    - APIキーはサーバーレス機能（APIプロキシ）としてサーバー側で安全に管理し、利用者はAPIキーの準備をすることなく機能を利用できるものとする。
- **役割:**
    - 専用のシステムプロンプトで役割を定義し、「Strudel専門家」として振る舞わせる。ユーザーの自然言語による指示から、適切なStrudelコードを生成する。

### 2.2. コードテンプレート機能

- **UI:**
    - 専用のフローティングボタンを配置し、クリックでモーダルウィンドウを開く。
- **機能:**
    - モーダル内には「ファンク」「テクノ」「アンビエント」といった音楽ジャンルのリストを表示する。
    - ジャンルを選択すると、あらかじめ用意された高品質なStrudelコードスニペットが表示される。
    - 「コピー」ボタンにより、表示されたコードをクリップボードにコピーできる。
- **目的:**
    - ユーザーが曲作りのインスピレーションや出発点として利用できる、静的なコード集を提供する。この機能はAIとは完全に独立している。
